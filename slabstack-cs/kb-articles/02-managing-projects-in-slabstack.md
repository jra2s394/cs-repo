# Managing Projects in Slabstack

Track every job from quote through pour completion in a single project record.

## Overview

A project in Slabstack is the container for everything related to one job: the customer, the quote, deliveries, communication, and status. Use projects to know exactly where each job stands without digging through email or your dispatch system.

## Step-by-Step

1. From the dashboard, click **Projects → New Project**.
2. Enter the project name. Use a naming convention your team can scan — e.g., `Customer – Job Site – Start Date`.
3. Assign the customer. If they don't exist yet, create them inline.
4. Set the project address. This is what shows on the quote and delivery tickets.
5. Choose the project type (Residential, Commercial, Infrastructure, Other) — this drives reporting later.
6. Set the expected start date and the estimated total quantity.
7. Assign a project owner — the person on your team responsible for this job.
8. Click **Create Project**.
9. From inside the project, use the tabs to navigate: **Overview**, **Quotes**, **Deliveries**, **Communication**, **Files**.

## Tips

- Status badges (Quoted, Active, Paused, Complete) live on every project. Filter the Projects list by status to run your daily standup off one screen.
- Use the **Communication** tab as the single source of truth — every email sent from Slabstack and every internal note ends up here.
- When a project closes, move it to **Complete** rather than deleting it. Completed projects still show in reports and let you reuse the data for the next job at that site.
